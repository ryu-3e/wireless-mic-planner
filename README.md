# ワイヤレスマイク Ch プランナー (B帯) — PWA

RAMSA (Panasonic) / Sony 等の **B帯 (806.125〜809.750 MHz, 30波)** ワイヤレスマイクの
多チャンネル運用を、混信や IMD (相互変調歪) を自動チェックしながら計画できる
オフライン対応の **PWA (Progressive Web App)** です。

## 機能

- 送信機インベントリ管理（**アナログ/デジタル方式の区別**）
- チャンネル手動割当（アナログ B-1〜B-7 / RAMSAデジタル 31/32/33 グループ対応）
- **自動チャンネル割当**（同一グループ優先 → IMDバックトラック）
- **クイック設計モード**（機材リストを使わずに即時設計）
  - 必要な波数（アナログ・デジタル）を指定
  - 会場で **既に使われている周波数を入力 → 空きチャンネルから自動割当**
  - 固定したいチャンネルを最大2つまで指定可能
  - 結果を機材インベントリに反映するボタンあり
- **IMD (3次相互変調) 検出**（2波 / 3波）
- **混信チェック**（同一周波数 / 占有帯域重なり / 隣接周波数）
- **アナログ・デジタル混在時のチャンネルプラン警告**
  - アナログ↔アナログ : 推奨最小 250 kHz
  - デジタル↔デジタル : 推奨最小 **375 kHz** (B帯 占有192kHz想定)
  - アナログ↔デジタル : 推奨最小 250 kHz
- 周波数軸バーチャート可視化（占有帯域幅も半透明矩形で表示）
- プリセット保存／読み出し（LocalStorage）
- PDF 出力（印刷経由 / 「PDFとして保存」を選択）
- CSV ダウンロード／クリップボードコピー
- **オフライン動作** （Service Worker でキャッシュ、ホーム画面追加で「アプリ化」）

## 対応機材（初期インベントリ）

本アプリでは **送信機 (TX) のみ** を扱います（チャンネル割当の対象は送信機側のため、受信機は表示しません）。

| メーカー | モデル | 方式 | 占有帯域 | 数 |
| --- | --- | --- | --- | --- |
| Panasonic | WX-RB400 | アナログ | 110 kHz | 1 |
| Panasonic | WX-TB840 | アナログ | 110 kHz | 4 |
| Panasonic | WX-TB841 | アナログ | 110 kHz | 2 |
| Panasonic | WX-DT135 | **デジタル** | 192 kHz | 2 |
| Sony | UTX-B40 | アナログ | 110 kHz | 2 |
| Sony | UTX-B40 (UWP-D21) | アナログ | 110 kHz | 1 |

合計 12 波（アナログ 10 / デジタル 2）。

## 起動方法

### 1. PC で動かす（最も簡単）

`start.bat` をダブルクリック → ブラウザで `http://localhost:8000/` を開く

> Python 3 もしくは Node.js が必要です。どちらも無い場合はインストール後に再実行してください。

### 2. スマホで使う

最終的にはスマホで使うのが本来の目的です。次のいずれかで配信してください。

#### A) ローカル LAN 経由 (一番手軽)

1. PC で `start.bat` を起動
2. PC の IPv4 アドレスを確認 (例: `ipconfig` で `192.168.x.x`)
3. スマホ (PC と同じ Wi-Fi) で `http://192.168.x.x:8000/` を開く
4. **Chrome なら右上メニュー → 「ホーム画面に追加」** でアプリのように起動可能

> ⚠ Service Worker は HTTP では一部機能制限あり。ローカル LAN なら動きますが、
> 完全な PWA 化には HTTPS が必要です（次の B 案を推奨）。

#### B) GitHub Pages 等の静的ホスティングへ配置 (推奨・無料)

1. このフォルダごと GitHub にリポジトリとして push
2. リポジトリ設定で「Pages」を有効化
3. 公開された `https://<user>.github.io/<repo>/` をスマホで開く
4. ホーム画面に追加 → **オフラインでも完全動作**

#### C) Netlify / Vercel / Cloudflare Pages にドラッグ&ドロップ

フォルダをそのままアップロードするだけで HTTPS で公開されます (無料枠で十分)。

## 使い方

### A) 自分の機材を管理しながら設計する場合

1. **機材** タブで送信機の所有数を確認・編集
2. **割当** タブで送信機ごとにチャンネルを選ぶ。または「⚡ 自動割当」を押して全自動で決定
3. **マップ** タブで周波数軸上の配置を視覚確認、IMD/隣接干渉の警告を確認
4. **メニュー** タブで現場ごとのプリセット保存、PDF/CSV 出力

### B) 現場で素早く設計する場合（クイック設計）

ホールやスタジアム等、その場で「7波必要、うちデジタル2波」「会場のB22は既に使用中」のような状況に対応。

1. **クイック** タブを開く
2. **アナログ波数 / デジタル波数** を ＋−ボタンで指定（例: アナログ5 + デジタル2）
3. （任意）**「使用中チャンネルを追加」** で会場で既に使われている周波数を登録
4. （任意）**「固定チャンネルを追加」** で必ず使いたいチャンネルを最大2つまで指定
5. **「⚡ 自動割当を計算」** をタップ → 干渉のない組合せを提案
6. 必要なら **「機材インベントリに反映」** で結果を保存

**自動割当が考慮するもの**:
- 使用中・固定の周波数の占有帯域 (アナログ110kHz / デジタル192kHz)
- 推奨最小間隔 (アナ↔アナ 250kHz / アナ↔デジ 250kHz / デジ↔デジ 375kHz)
- IMD3次 (2波・3波)
- 同一グループ優先（全アナログの場合は B-1〜B-7 から選択）

### グループの最大同時波数

**アナログ B帯 (Sony 4-530-738-01 準拠 / 最小 250 kHz 間隔)**:
- B-1, B-2, B-3, B-4 = 各 **6 波** (例: B-1 = B11/B12/B13/B14/B15/B16)
- B-5 = **5 波** (B51〜B55)
- B-6 = **1 波** (B61)
- B-7 = **7 波** (B11/B12/B33/B36/B52/B54/B55) — B-8 と **択一**
- B-8 = **7 波** (B21/B31/B13/B14/B25/B16/B46) — B-7 と **択一**

> ※ B-7 と B-8 は「同一エリアでは同時に使えない」相互排他のグループ。7波必要なときはどちらか一方を選ぶ。

**RAMSA デジタル B帯（31, 32, 33、375 kHz 等間隔）**:
- 各グループ **10 波** (例: グループ31 = BF1/BE1/BF2/BE2/BF3/BE3/BF4/BE4/BF5/BE5)
- デジタルは同じグループ内なら 10 波同時運用が可能
- B帯30波が3グループに均等分散（各10波）

> 参考資料: **Sony 周波数リスト 4-530-738-01 (B型 / グループ ALL, B-1〜B-8)** / JVC WT-900 シリーズ周波数表 / Panasonic WX-TB841 取扱説明書 / Panasonic WX-DT135 仕様書 / **Panasonic WX-DR131 周波数表(800MHz帯) PGQW1833ZAJ1** / Sony UWP-D21 仕様書 / [デジタルB帯のワイヤレスマイクについて (note記事)](https://note.com/tkykmts/n/n5d8cfdf02bc7)

## アナログB帯 と デジタルB帯 の違い

| 項目 | アナログB帯 | デジタルB帯 |
| --- | --- | --- |
| 占有周波数帯幅 | 約 110 kHz | 約 **192 kHz** |
| 同時運用本数(目安) | 6〜7 波 | 最大 10 波程度 (機種・メーカーによる) |
| 推奨チャンネル間隔 | グループ表に従う (最小 250 kHz) | **375 kHz 等間隔** が主流 |
| 遅延 | 実用上ゼロ | 1〜3 ms 程度 |
| 干渉耐性 | 標準 | アナログより強い |

**混在運用の注意**:
アナログ用チャンネルプランをそのままデジタルに適用すると、占有帯域幅が広い分干渉する場合があります。
本アプリでは方式が異なるペアの隣接距離も自動チェックし、警告を出します。
